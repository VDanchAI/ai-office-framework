# Структура брендбука

> Брендбук — это аддон, который подключается к ролям, работающим с контентом и коммуникацией.
> Загружается в: Копирайтер, Маркетолог, Дизайнер.

---

## Модульная архитектура

Брендбук — это не один документ. Это **система из 4 модулей**, каждый со своим роутером, протоколами и секциями. Плюс архитектурный файл, описывающий как модули связаны.

```
Brandbook/
├── BRANDBOOK_MODULAR       ← Архитектура: как модули связаны, иерархия, принципы
├── BRAND_IDENTITY          ← КТО: манифест, ядро бренда, аудитория, методология
├── BRAND_VOICE             ← КАК ГОВОРИТ: тон, стиль, словарь, запреты, few-shot
├── BRAND_VISUAL            ← КАК ВЫГЛЯДИТ: цвета, шрифты, портрет, принципы
└── BRAND_PLATFORMS         ← ГДЕ ПУБЛИКУЕТ: форматы площадок, диспетчер, протоколы
```

### Связь между модулями

Модули работают не изолированно — они ссылаются друг на друга:

- **Identity** задаёт фундамент: кто бренд, для кого, в чём суть
- **Voice** строит стиль речи на основе Identity
- **Visual** задаёт визуальный язык, согласованный с Identity
- **Platforms** адаптирует Voice и Visual под конкретные площадки

---

## Модуль 1: Identity (Идентичность)

**Что определяет:** Кто этот бренд, для кого он, в чём его уникальность.

| Секция | Содержание |
|--------|-----------|
| Манифест | Неизменяемое ядро — миссия, видение, основной принцип |
| Ядро бренда | УТП, философия, слоган, архетип |
| Целевая аудитория | Портреты, боли, антипортреты (кто НЕ клиент) |
| Методология | Как бренд решает проблемы клиентов |

**Приоритет:** CRITICAL — ни один другой модуль не может противоречить Identity.

---

## Модуль 2: Voice (Голос бренда)

**Что определяет:** Как бренд говорит — тон, стиль, словарь, интонация.

| Секция | Содержание |
|--------|-----------|
| Тон и стиль | Правила тона для разных контекстов (продажи, обучение, поддержка) |
| Запретный словарь | Слова и фразы, которые бренд НИКОГДА не использует |
| Few-shot примеры | Эталонные фрагменты текста «так звучит этот бренд» |
| Customer Journey | Как тон меняется на разных этапах пути клиента |
| Чек-лист | Проверка текста на соответствие голосу перед публикацией |

**Ключевой принцип:** Бот пишет голосом владельца бренда, а не обезличенным «тоном ИИ».

---

## Модуль 3: Visual (Визуальная идентичность)

**Что определяет:** Как бренд выглядит — цвета, шрифты, стиль изображений.

| Секция | Содержание |
|--------|-----------|
| Визуальные принципы | Правила, которым подчиняется весь визуал |
| Палитра | Основные и акцентные цвета с кодами |
| Типографика | Шрифты, размеры, иерархия |
| Портрет | Фирменный визуальный стиль (фото, иллюстрации, графика) |

---

## Модуль 4: Platforms (Платформы)

**Что определяет:** Как контент адаптируется под каждую площадку.

| Секция | Содержание |
|--------|-----------|
| Форматы площадок | Требования к контенту: Telegram, Instagram, сайт, email, YouTube |
| Диспетчер форматов | Маршрутизация: какой формат для какой площадки |
| Гейты качества | 6 проверочных точек перед публикацией |
| Аудио/Видео протоколы | Правила для голосового и видеоконтента |

---

## Внутренняя архитектура каждого модуля

Все модули следуют единой структуре из 6 блоков:

```
BLOCK 1: Философия модуля     ← Назначение, центральный принцип, связи
BLOCK 2: JSON Router           ← Навигация по секциям модуля
BLOCK 3: Назначение            ← Детальное описание задач
BLOCK 4: AI Priority           ← Жёсткие правила (Hard Rules)
BLOCK 5: Протоколы             ← Как бот работает с этим модулем
BLOCK 6: Техническая спека     ← Форматы, ограничения, стандарты
```

Каждый модуль имеет свой JSON-роутер — бот загружает только нужную секцию, а не весь модуль.

## Как брендбук работает в системе

1. Брендбук загружается как **Add-on** в Claude Project нужной роли
2. Копирайтер получает Voice + Identity → пишет текст голосом бренда
3. Маркетолог получает Identity + Platforms → строит стратегию с учётом позиционирования
4. Дизайнер получает Visual + Identity → оформляет в фирменном стиле

**Один брендбук — все роли говорят одним голосом.**

---

## Кастомизация

Брендбук кастомизируется под каждого клиента:
- Identity заполняется с нуля под конкретный бренд
- Voice настраивается на живой голос владельца (few-shot из реальных текстов)
- Visual описывается на основе существующего стиля или создаётся новый
- Platforms адаптируются под реальные каналы клиента
